# Integração — S3 (armazenamento de imagens)

> Código: `src/shared/integrations/s3` · Consumidores: `media`, `content`

## Visão geral

Armazenamento de objetos compatível com S3 (AWS S3, Cloudflare R2, MinIO em
dev). Configuração via envs `S3_*` (ver `.env.example`).

## Organização do bucket

```
contests/<ano>/frame.png                              # moldura da edição
contests/<ano>/registrations/<registrationId>/<uuid>.<ext>  # fotos
partners/<partnerId>/logo.<ext>                       # logos
blog/<postId>/cover.<ext>                             # capas do blog
```

## Princípios

1. **Banco guarda só a `storageKey`** — nunca URL completa. URL pública é
   montada com `getPublicUrl()` (env `S3_PUBLIC_URL`, idealmente CDN).
2. **Upload direto do browser** via presigned URL (validade 5 min) — a imagem
   nunca passa pelo servidor Next (evita limite de body e custo de banda).
3. Bucket privado para escrita; leitura pública apenas via CDN/dist. Fotos de
   inscrições **não aprovadas** não são linkadas em lugar nenhum público.
4. Keys usam UUID — sem dados pessoais no caminho além do id interno.
5. Deleção: ao trocar foto ou rejeitar inscrição com troca, remover objeto
   órfão (`deleteObject`).

## API do client

| Função | Descrição |
| --- | --- |
| `buildPhotoKey({contestYear, registrationId, fileName})` | Gera key padronizada |
| `createPresignedUploadUrl(key, contentType)` | URL de upload (PUT, 5 min) |
| `getPublicUrl(key)` | URL pública via CDN |
| `deleteObject(key)` | Remove objeto |

## Dev local

Usar MinIO (`S3_ENDPOINT=http://localhost:9000`) — o client ativa
`forcePathStyle` automaticamente quando há endpoint customizado.
